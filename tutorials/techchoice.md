# Teknologivalg og begrundelse

## Formål
Formålet med teknologivalget er at sikre en hurtig udviklingsproces, en overskuelig kodebase og en stabil løsning til håndtering af data.  

Valgene er delvist baseret på anbefalinger fra undervisere, men også på hvad der er praktisk og passende for projektets omfang.

---

## Frontend

### Værktøjer
| Teknologi | Formål |
|----------|--------|
| Vite | Build tool og development server |
| Vue | Frontend framework |
| SCSS | Styling |

---

### Vite

**Hvad er det?**  
Vite er et moderne build tool, der bruges til at starte og bygge frontend applikationen.

**Begrundelse:**
- Meget hurtig opstart og udviklingsserver  
- Understøtter hot module replacement (HMR)  
- Simpel opsætning sammenlignet med fx Webpack  
- God integration med Vue  

---

### Vue

**Hvad er det?**  
Vue er et JavaScript framework til at opbygge brugergrænseflader.

**Begrundelse:**
- Komponentbaseret struktur (genbrugelig kode)  
- Let at lære og arbejde med  
- God dokumentation  
- Velegnet til mindre og mellemstore projekter  

---

### SCSS

**Hvad er det?**  
SCSS er en udvidelse af CSS med ekstra funktioner.

**Begrundelse:**
- Understøtter variabler og nesting  
- Gør styling mere struktureret  
- Nem at vedligeholde i større projekter  

---

## Backend og database

### Værktøjer
| Teknologi | Formål |
|----------|--------|
| Firestore | Database til strukturerede data |
| Realtime Database | Database til realtidsdata |

---

### Firebase

**Hvad er det?**  
Firebase er en cloud-baseret platform, der tilbyder backend funktionalitet uden behov for egen server.

**Begrundelse:**
- Ingen opsætning af backend nødvendig  
- Nem integration med frontend  
- Skalerbar løsning  
- Anbefalet i undervisningen  

---

### Firestore

**Hvad er det?**  
Firestore er en NoSQL database fra Firebase.

**Begrundelse:**
- Velegnet til strukturerede data  
- Understøtter forespørgsler  
- Fleksibel datamodel  

---

### Realtime Database

**Hvad er det?**  
Realtime Database er en database, der opdaterer data live.

**Begrundelse:**
- Data opdateres i realtid  
- God til simple og dynamiske data  
- Let at implementere  

---

## Overvejelser

Valget af teknologier er påvirket af undervisernes anbefalinger, hvilket har gjort det nemmere at komme i gang med projektet.  

Samtidig er der valgt teknologier som:
- er lette at arbejde med  
- har god dokumentation  
- understøtter hurtig udvikling  

---

## Mulige alternativer

Eksempler på alternativer kunne være:
- Node.js + Express (backend)  
- SQL database i stedet for NoSQL  
- React i stedet for Vue  

Disse er ikke valgt for at reducere kompleksitet og fokusere på projektets funktionalitet.
